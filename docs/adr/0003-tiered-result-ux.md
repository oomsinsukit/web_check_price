# Tiered Result: ระบบไม่ฟันธงราคาเมื่อไม่มั่นใจ — ให้ผู้ใช้เป็นคนยืนยันขั้นสุดท้าย

plush รุ่นคล้ายกันมีเป็นสิบแบบ (ท่าทาง/ป้าย/ไซส์ต่าง ราคาต่างกันหลายเท่า) AI ระบุรุ่นเป๊ะๆ จากรูปเดียวไม่ได้อย่างน่าเชื่อถือ เราจึงออกแบบผลลัพธ์เป็นสองชั้น: มั่นใจสูง → แสดง Price Summary ของสินค้าชิ้นนั้นตรงๆ / มั่นใจต่ำ → แสดง grid ของ Sold Listings ที่คล้ายพร้อมช่วงราคา ให้ผู้ใช้จิ้มตัวที่ตรงกับของในมือ (Match Confirmation) แล้วค่อยสรุปราคาเฉพาะรุ่นนั้น

นี่คือกลไกหลักที่ทำให้ "เงินน้อย + แม่นสูง" อยู่ด้วยกันได้: ความแม่นขั้นสุดท้ายมาจากตาผู้ใช้ ไม่ใช่จากโมเดลแพงๆ และระบบซื่อสัตย์ ("นี่คือของที่คล้ายกัน") แทนการเดาราคาผิดรุ่นแบบเนียนๆ ซึ่งทำลายความน่าเชื่อถือ

## Considered Options

- ฟันธงตัวเลขเดียวเสมอ: UX เรียบแต่โกหกโดยธรรมชาติ — ปฏิเสธ
- บังคับให้พิมพ์ชื่อรุ่นก่อนแสดงราคา: แม่นแต่ friction สูง ผู้ใช้ส่วนใหญ่ไม่รู้ชื่อรุ่นอยู่แล้ว — ปฏิเสธ
